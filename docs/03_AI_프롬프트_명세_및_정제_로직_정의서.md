# AI 프롬프트 명세 및 정제 로직 정의서

## 1. 다양한 이력서 포맷의 일관성 있는 처리 로직 (올바른 처리를 위한 검토안)

이력서는 자유 형식이므로, 단순 규칙 기반(Rule-based) 파싱으로는 정확한 정보를 추출하기 불가합니다. 다음 단계를 거쳐 DB에 저장성을 보장합니다.

1. **포맷별 전문 파서(Parser) 사용 (Python 활용 우선 고려):**
   - `.pdf`: `PyMuPDF`, `pdfplumber` (표 구조 인식 유리)
   - `.docx`: `python-docx`
   - `.hwp`: `pyhwp`, `hwp5txt` (한글 문서 전용 추출기)
   - `.xlsx`: `openpyxl`
2. **텍스트 정제 (Pre-processing):**
   - 불필요한 공백, 특수문자, 깨진 문자를 1차적으로 필터링.
   - 텍스트가 너무 긴 경우(토큰 한도 초과), 의미 단위(경력, 학력 등)로 청크 분할.
3. **AI (LLM) 기반의 구조적 추출 (Structured Output):**
   - OpenAI의 Function Calling 또는 JSON Mode, 혹은 Gemini의 Structured Outputs 기능을 사용하여 AI가 반드시 우리가 원하는 JSON 키와 타입(문자열, 숫자, 배열)으로 응답하도록 강제합니다.
4. **후속 검증 (Validation layer):**
   - 추출된 JSON이 DB 스키마(이름 존재 여부, 연차 Number 타입 체크 등)에 부합하는지 Pydantic 클래스나 DTO를 통해 유효성 검사를 수행합니다.

## 2. AI 시스템 프롬프트 명세서

**[System Message]**
```text
당신은 이력서 데이터를 분석하고 정형화된 JSON 포맷으로 추출하는 전문 HR Assistant AI입니다.
주어진 텍스트는 지원자의 이력서 내용입니다.
이 내용을 바탕으로 아래 규칙에 따라 반드시 정확한 JSON 객체 형태로만 답변해야 합니다.
지원자의 텍스트에 내용이 없다면 null 또는 빈 배열([])로 처리하세요. 존재하지 않는 내용을 유추해서 지어내면 절대 안됩니다.
```

**[User Message (Input)]**
```text
이력서 내용: {추출된 원본 텍스트 데이터}

[추출 규칙]
1. `name`: 지원자 이름 (문자열)
2. `birthdate`: 생년월일 (YYYY-MM-DD 형식. 연도만 있으면 YYYY-01-01으로 처리)
3. `yearsOfExperience`: 총 연차 (정수형. 신입이면 0)
4. `techStacks`: 사용 기술 스택 (문자열 배열)
5. `career`: 경력 사항 요약 (문자열 배열, 각 직장별 업무 요약)
6. `osExperience`: 사용 경험이 있는 운영체제 (문자열 배열, ex: Linux, Windows, macOS)
7. `languages`: 사용 가능한 외국어 및 수준 (문자열 배열)
8. `skills`: 기타 직무 관련 스킬 (문자열 배열)

출력은 반드시 다른 부가적인 텍스트(마크다운 블록 등) 없이 순수 JSON 포맷으로만 반환하세요.
```

## 3. 정제 프로세스 에러 처리 (Fallback Strategy)
- 환각 현상(Hallucination)으로 필드가 누락된 경우, 재시도(Retry) 로직 (최대 3회) 적용.
- 실패한 문서는 `Parsing Failed` 상태로 관리자 모니터링 메뉴에 표시되어, 수동 기록이 가능하도록 UX 제공.
