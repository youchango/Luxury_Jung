# 개발 환경 및 기술 스택 정의서 (Luxury-Jung)

## 1. 프론트엔드 (사용자 및 관리자 UI)
풍부한 검색 조건과 고성능 조회가 필요한 복합 UI를 구현합니다.
- **Framework:** **Next.js (React 기반)** 또는 **React.js (Vite)**
  - 컴포넌트 단위 관리 및 렌더링 최적화.
- **UI 라이브러리:** **TailwindCSS** + **Shadcn UI** 또는 **MUI (Material UI)**
  - 대량의 필터(아코디언, 체크박스, 슬라이더)와 데이터 테이블 구현에 유리.
- **State Management:** **Zustand** 또는 **React Query (TanStack Query)**
  - 페이징, 필터 상태 유지 관리.

## 2. 메인 백엔드 (API 및 데이터 서빙)
사용자 인터페이스와 빠르고 안정적인 API 통신을 담당합니다.
- **Framework:** **Java 17 (이상) + Spring Boot 3.x**
  - 복잡한 도메인(Auth, Search, API Gateway 역할) 관리 및 안정성 확보.
- **ORM:** **Spring Data JPA & QueryDSL**
  - 대용량 데이터의 동적 쿼리(복잡한 이력서 검색 조건 적용) 구현에 강력함.
- **Authentication:** **Spring Security + JWT**

## 3. 파싱 에이전트 & AI 통신부 (첫번째 프로그램)
이력서 파일 처리(특히 한글 HWP 등) 라이브러리 지원이 강력한 Python을 전담 워커로 사용합니다.
- **Framework:** **Python 3.10+ & FastAPI**
- **Task Queue:** **Celery + Redis/RabbitMQ** (비동기 파일 처리 대기열)
- **Data Validation:** **Pydantic** (AI가 내뱉은 JSON 결과의 스키마 유효성 검증)
- **AI API:** **OpenAI API (gpt-4o) / Anthropic Claude / Gemini API** (선택)
- **File Parsers:** `pdfplumber`, `python-docx`, `pyhwp`, `openpyxl` 등

## 4. 데이터베이스 및 인프라
대용량 데이터의 지연 없는 검색과 원본 보존을 모두 만족하는 구조입니다.
- **RDBMS:** **PostgreSQL**
  - JSONB 타입을 지원하여 확장성 있는 AI 분석 결과 저장에 탁월.
- **Search Engine:** **Elasticsearch** (권장, 옵션사항)
  - "보유 스킬 중 Java 포함이면서 3년차 이상" 과 같은 복합, 전문(Full-text) 검색 시 빠른 응답성(100ms 이하) 보장. RDBMS만으로 Paging 속도 저하가 있을 경우 도입 필수.
- **File Storage:** **AWS S3** 또는 로컬 기반 MinIO (원본 이력서 파일 객체 저장).
